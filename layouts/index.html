go-embedded-ruby

A pure-Go implementation of Ruby — one static binary, full dynamism, zero cgo.

pure Go · zero cgo single static binary bytecode VM embedded front-end (eval/require) reuses Go's GC Ruby 4.0 semantics
Documentation GitHub

go-embedded-ruby compiles Ruby to bytecode and runs it on a stack VM in the mruby/YARV lineage, with the lexer, parser and compiler embedded in the binary so eval and runtime require keep working. Ruby objects are Go heap objects, so Go's garbage collector is reused — there is no GC to write. Because it is pure Go with cgo disabled it cross-compiles trivially, and it is the first Ruby you can embed inside a Go program with no C toolchain.

Pure Go with cgo disabled, so it cross-compiles to a single static binary. A bytecode VM with the front-end embedded keeps Ruby's dynamism — eval, runtime require, monkey-patching, method_missing — at your fingertips. Ruby objects are Go heap objects, so the Go garbage collector is reused. The standard library is selected at build time and tree-shaken. Semantics track Ruby 4.0, grown test-first against an MRI differential oracle. The regexp engine is a pure-Go reimplementation of Onigmo, developed in the sibling org github.com/go-onigmo.
